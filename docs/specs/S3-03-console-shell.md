# S3-03 后台壳层与权限路由规格

版本日期：2026-06-10

## 1. 目标

建立 HeOS `/console` 后台入口，把 `S2-04` 服务端访问上下文与菜单查询函数接入实际页面。该任务交付可访问、可验证、可继续扩展的后台壳层，为后续租户权限、资产管理、监测告警、农事追溯和系统管理页面提供导航框架。

## 2. 范围

本任务覆盖：

- 新增 `/console` 路由。
- 后台壳层使用服务端菜单结果渲染导航。
- 页面展示当前访问摘要：租户、数据范围、角色、权限数量。
- 页面展示后续模块入口和权限状态。
- 首页公开 demo 保持不变。
- 未登录访问 `/console` 继续使用现有登录保护。
- 登录成功后进入 `/console`。

本任务不覆盖：

- 租户、用户、角色 CRUD。
- 真实 D1 查询。
- 复杂数据表格。
- 业务详情页。
- OIDC 登录替换。

## 3. 交互约束

- 桌面端采用左侧导航、顶部状态区、主内容区。
- 移动端导航横向滚动或折叠到顶部区域，不产生页面横向溢出。
- 导航项来自 `getCurrentConsoleMenu`，客户端不自行构造权限菜单。
- 当前阶段模块卡片作为后续入口，不承担真实业务写入。

## 4. 验收标准

- `/console` 可作为登录后后台入口。
- `/console` 展示服务端返回的菜单。
- `/console` 展示租户、角色、数据范围、权限数量。
- 登录页登录成功后跳转 `/console`。
- 首页 `/` 继续作为公开 demo。
- `pnpm test` 通过。
- `pnpm build` 通过。
- 桌面端和移动端浏览器检查通过。
