# S3-05 控制台主业务页面

## 1. 范围

本规格覆盖 `/console` 工作台内的 PRD 主业务页面入口和首批只读页面区块：

- 项目资产
- 设备台账
- 作物模型
- 农事任务
- 告警中心
- 追溯档案
- AI 辅助记录

页面先以工作台区块承载，入口通过锚点进入对应业务区块。后续进入 CRUD 阶段时，再按权限和 URL 结构拆分独立子路由。

## 2. 数据来源

- 项目资产、设备台账、农事任务、告警中心、追溯档案和 AI 辅助记录来自 S2-05 核心域查询 handler。
- 作物模型来自一期核心域模型的演示数据结构，后续切换为 D1 查询。
- 追溯档案在工作台层过滤 `visibility = public`，不展示非公开批次。

## 3. 交互与展示

- 工作台提供 7 个主业务入口。
- 设备台账展示 `siteId` 与 `limit` 筛选条件、总数和 `nextCursor`。
- 告警中心展示 `status = open` 筛选条件、总数和 `nextCursor`。
- 农事任务展示 `status = planned` 筛选条件、总数和 `nextCursor`。
- 追溯档案展示 `visibility = public` 条件。
- 每个业务区块保留空状态文案。
- 移动端使用单列布局，长文本允许换行。

## 4. 验收

- 登录后从 `/console` 能进入 7 个主业务区块。
- 页面显示 S2-05 服务端查询结果。
- 设备、告警、农事任务显示筛选条件和分页游标。
- 追溯档案只显示公开记录。
- 桌面与移动端无横向溢出。
- `pnpm test` 通过。
- `pnpm build` 通过。
