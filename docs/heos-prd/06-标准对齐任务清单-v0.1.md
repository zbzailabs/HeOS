# HeOS 标准对齐任务清单 v0.1

版本日期：2026-06-09

## 1. 任务拆分原则

- 每条任务必须绑定 `06-标准对齐规格-v0.1.md` 的条目。
- 每条任务必须有可复现验收命令或接口校验点。
- 任务优先级默认按 `P0`（阻塞）> `P1`（交付）> `P2`（优化）。
- 未完成项不允许进入发布冻结。

## 2. 任务看板（待建）

### P0. 数据标准与字典

1. `S0-01` 建立标准术语与单位字典表
   - 关联项：Spec 4.1 标准化数据字典，Spec 5.1 数据模型
   - 交付：`StandardDictionary` 字典基础集（作物、生育期、指标、设备能力、告警级别）
   - 验收：
     - 字典条目具备 `code/version/source/effectiveFrom/effectiveTo` 字段
     - 字段变更写入变更记录
   - 预估：1 天

2. `S0-02` 建立跨模块枚举统一文件
   - 关联项：Spec 3.1 指南结构映射
   - 交付：`MetricCode`、`AlertType`、`AlertLevel`、`SyncStatus` 等统一枚举
   - 验收：重复的枚举值仅允许单源定义
   - 预估：0.5 天

3. `S0-03` 建立权限管理与后台管理壳层规格
   - 关联项：Spec 4.1 安全与可信、Spec 5.1 数据模型、PRD 4.1 租户与权限
   - 交付：`Tenant`、`OrgUnit`、`User`、`Role`、`Menu`、`Permission`、`DataScope`、`AuditLog` 的一期边界，`/console` 后台壳层边界，权限路由和服务端菜单过滤边界
   - 参考：RuoYi 的权限管理模型，soybean-admin 的后台布局和权限路由模式
   - 验收：明确参考范围和排除范围，后续实现不迁移 Java/Spring、Vue、NaiveUI、UnoCSS 技术栈
   - 预估：0.5 天

### P0. 数据模型与迁移

4. `S1-01` 实现 `TelemetryLatest` 与 `TelemetryHistory`
   - 关联项：Spec 5.1 数据模型
   - 交付：
     - `tenantId/siteId/deviceId/metricCode` 唯一约束
     - `TelemetryHistory` 复合索引
     - 幂等写入策略
   - 验收：
     - 重复采样入库无重复主键错误
     - 历史查询返回按时间升序/降序稳定分页
   - 预估：2 天

5. `S1-02` 实现 `Alert` 与 `StandardRule` 规范表
   - 关联项：Spec 4.1 告警规则标准化，Spec 6.3
   - 交付：告警原因、触发值、阈值、处置动作可回溯
   - 验收：
     - 告警规则字段与接口返回一致
     - 离线告警与阈值告警可并行存在不重复失效
   - 预估：1.5 天

6. `S1-03` 建立 `AuditLog` 标准字段扩展
   - 关联项：Spec 4.1 审计与合规日志，Spec 6.3
   - 交付：审计字段标准模板 `traceId/userId/tenantId/action/target/result/latencyMs`
   - 验收：
     - 登录、导出、查询、控制申请、同步事件全部落库
     - 关键事件可按租户和时间检索
   - 预估：1 天

### P1. 同步与接口

7. `S2-01` 标准化 Renke 同步 API
   - 关联项：Spec 5.2 供应商同步入口
   - 交付：
     - `renke` token 刷新策略
     - 采样标准化映射 `metricCode/unit/quality`
     - 同步摘要返回 `total/updated/failed/ts`
   - 验收：
   - `AUTH_TIMEOUT`、`SCHEMA_MISMATCH`、`SOURCE_TIMEOUT` 可区分处理
   - 同步失败记录 `failed` 计数
   - 预估：2 天

8. `S2-02` 建立遥测查询 API
   - 关联项：Spec 5.2 遥测查询
   - 交付：
     - `GET /api/telemetry/latest`
     - `GET /api/telemetry/history`
     - 查询参数校验
   - 验收：
     - 缺失 tenant/device/metric 返回 400
     - 时间窗口校验合理（fromTs <= toTs，时间范围不超限）
   - 预估：1 天

9. `S2-03` 建立标准规则管理 API
   - 关联项：Spec 5.2 告警规则
   - 交付：
     - `POST /api/standards/rules`
     - `PATCH /api/standards/rules/{id}`
     - `GET /api/standards/rules/validation`
   - 验收：
     - 规则保存必须带版本号
     - 非法阈值拒绝写入
   - 预估：1 天

### P1. 页面与可视化

10. `S3-01` 在线状态与离线告警前端展示
   - 关联项：Spec 5.1 在线判定 + Spec 6.2
   - 交付：
     - 设备列表显示在线/离线
     - 告警列表标注离线原因和触发时间
   - 验收：
     - 5 分钟内未上报判定离线
     - 告警关闭/恢复记录完整
   - 预估：1 天

11. `S3-02` 标准稽核页
   - 关联项：Spec 5.2 标准稽核、Spec 6.1、7
   - 交付：
     - `GET /api/compliance/checklist` 前端展示
     - 按指南条目列出覆盖与缺口
   - 验收：
     - 存在未覆盖关键项时给出阻断标记
     - 页面可导出当前快照
   - 预估：1.5 天

### P2. 运营与治理

12. `S4-01` 发布冻结前 `compliance-report` 产出
   - 关联项：Spec 6.1、7、9
   - 交付：
     - `compliance-report` 模板
     - 未达标项清单、原因、修复计划、下次迭代目标
   - 验收：
     - 每次发布附报告
     - 报告中无“未评估阻塞项”遗漏
   - 预估：0.5 天

13. `S4-02` 任务边界与验收命令固化
   - 关联项：Spec 1、6、9
   - 交付：
     - 任务边界与验收命令清单
     - 与 `docs/HeOS 产品需求文档20260608.md` 保持一致
   - 验收：
     - 复用命令 `pnpm test`、`pnpm build`、标准验收 checklist
   - 预估：0.5 天

## 3. 里程碑与责任分配

### 里程碑

- M1（D+5）：完成 P0 与 P1-数据模型任务，提交 schema 与迁移
- M2（D+10）：完成 P1 接口任务与 P2 标准稽核端能力
- M3（D+14）：完成页面联调、标准稽核、发布冻结产物

### 任务边界

- 本版本不含畜禽/渔业深度模型
- 本版本不含自动控制下发
- 外部 PostgreSQL/Neon 仅保留为评审后补充方案

## 4. 提交要求

每个任务完成后在代码提交中写入：
- 任务编号与文件清单
- 关键验证结果（含失败项）
- 未完成项与下一步建议
- 影响的标准条目编号
