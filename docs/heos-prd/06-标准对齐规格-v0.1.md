# HeOS 标准对齐 Spec v0.1

版本日期：2026-06-09

## 1. 目标与范围

本规格将农业农村部《智慧农业标准体系建设指南》作为一期 HeOS 的基础规范源，覆盖以下目标：

- 以指南三层结构（基础通用、关键技术、应用领域）对 HeOS 一期能力进行硬约束。
- 把第一条业务链路（腾龙小学智慧农场）作为标准实施试点。
- 形成从术语、数据、接口到验收的可复用标准清单。

适用范围：

- 版本范围：HeOS 一期，TanStack Start + Cloudflare-first（不含 Prisma/Neon 默认化）
- 技术范围：平台管理端、租户隔离、设备监测、告警、追溯、AI 辅助、异步同步
- 生效范围：首次发布前的开发、评审、验收均按本规格执行

## 4. 标准对齐功能包 v0.1（按最小闭环发布）

### 4.1 必须实现

- 标准化数据字典
  - 文件：`docs/heos-prd/06-标准对齐规格-v0.1.md`（本文件）
  - 任务：建立统一字典：作物、生育期、指标、设备能力、供应商状态、告警级别
  - 验收：表级字段与接口枚举出现一致性错误时，接口返回 `400` 并给出码表引用

- 遥测采集与标准化
  - 任务：定义 `TelemetrySample` 与 `TelemetryLatest` 统一模型
  - 关键字段：`ts`、`tenantId`、`siteId`、`deviceId`、`metricCode`、`metricValue`、`unit`、`quality`、`source`
  - 验收：同一采样时间同一指标重复到达时按主键幂等落库

- 在线状态判定
  - 规则：样本时间与当前时间差 > 300 秒判定离线
  - 验收：5 分钟离线判定在 `Alert` 与设备列表中可见

- 告警规则标准化
  - 任务：告警规则用统一结构 `ruleId/ruleType/threshold/lower/upper/effectiveFrom/effectiveTo/action` 表达
  - 验收：超过阈值产生告警并留痕 `createdBy=system-rule`

- 审计与合规日志
  - 覆盖事件：登录、导出、查询、控制申请、策略修改、供应商同步
  - 验收：`AuditLog` 记录字段完整，包含请求 trace、用户、租户、动作、结果、耗时

### 4.2 二期补齐

- 畜禽/渔业场景模型扩展
- AI 决策解释链路与策略追溯增强
- 标准验证服务平台化（可外部提交标准化稽核报告）

## 5. 技术规格（最小实现）

### 5.1 数据模型（v0.1）

- `TelemetryLatest`
  - 主键：`tenantId + siteId + deviceId + metricCode`
  - 约束：每设备每指标保留最近一条，`ts` 可回退仅更新最新时
  - 输出：`value`, `unit`, `ts`, `quality`, `isOnline`, `source`

- `TelemetryHistory`
  - 复合索引：`tenantId, siteId, deviceId, metricCode, ts`
  - 约束：按小时落盘分区时可按后续增长扩展
  - 保留策略：默认 30 天热数据可查，超过窗口进入归档流程

- `Alert`
  - 必填：`tenantId/siteId/deviceId/alertType/level/metricCode/threshold/valueObserved/createdAt/suggestedAction`
  - 规则：离线告警、阈值告警、供应商告警并行生效

- `StandardDictionary`
  - 统一存储术语、单位、版本、来源（guide/issue/team）
  - 变更流程：新增版本需附变更理由和生效时间

### 5.2 服务端接口（v0.1）

- 供应商同步
  - 入口：`/api/providers/renke/sync`
  - 行为：仅服务端调用，返回同步摘要 `total/updated/failed/ts`
  - 错误码：
    - `AUTH_TIMEOUT` 重试入队
    - `SCHEMA_MISMATCH` 标记需修订映射
    - `SOURCE_TIMEOUT` 标记供应商侧超时

- 遥测查询
  - `GET /api/telemetry/latest`
  - `GET /api/telemetry/history`
  - 参数需包含 `tenantId, deviceId, metricCode, fromTs, toTs`

- 告警规则
  - `POST /api/standards/rules`
  - `PATCH /api/standards/rules/{id}`
  - `GET /api/standards/rules/validation`

- 标准稽核
  - `GET /api/compliance/checklist`
  - 返回：指南映射项、覆盖状态、未覆盖项、阻断项

### 5.3 接口标准

- 时间统一使用 ISO 8601（毫秒）
- 数值单位按数据字典统一（温度、湿度、光照、土壤湿度、EC、ph）
- 成功码与失败码明确且可重放
- 所有外部接口响应必须带 `traceId`

## 6. 验收标准（对齐章节）

### 6.1 标准覆盖率

- 100% 已上线的基础通用和关键技术 API 必须有对应标准说明条目
- 任何无标准条目的接口，禁止进入生产分支

### 6.2 质量指标

- `在线状态准确率`：在线判断与实测同步间隔偏差控制在 ±15 秒
- `告警触发一致率`：规则匹配一次，告警不缺失不重复，重试期间仅保留一次当前状态
- `采样入库及时率`：Renke 采样 `2 分钟` 内至少一次落库成功
- `供应商失败恢复率`：异常回退到队列后 24 小时内恢复率达到 95%

### 6.3 可审计性

- 同步失败、权限变更、控制申请必须有日志和恢复处理记录
- 每次发布附 `compliance-report`，包含未达标项清单与下次修复目标

## 7. 实施节奏

|阶段|周期|输出|门禁|
|---|---|---|---|
|S0 需求映射|D1-D2|完成指南映射表与字典清单|映射条目有编号、责任人、计划完成日|
|S1 实施开发|D3-D10|遥测标准化、在线判定、告警规则、审计增强|关键测试通过 `pnpm test` 并补齐数据模型迁移|
|S2 联调验收|D11-D14|Renke 同步链路通过、告警闭环、追溯展示|不满足标准的项禁止发布|
|S3 发布冻结|D15|生成 `compliance-report`|发布需附验收日志与飞书记录链接|

## 8. 风险与约束

- 供应商数据字段变化导致映射失效：版本化字典更新，新增字段必须有 `unknown` 容错记录
- 多租户字段混入：所有查询必须携带租户上下文
- 高频时序超出 D1 当前窗口：由架构评审触发外部时序存储评估
- 人工阈值频繁更改：变更必须通过审批并生成标准修订记录

## 9. 变更与追踪

- 规则和字段变更必须在 `docs/heos-prd` 内登记，更新版本号：v0.1 -> v0.2。
- 每次任务结束需在飞书项目同步：
  - 修改文件清单
  - 完成项/未完成项
  - 验证命令与结果
  - 下一步标准补齐项
